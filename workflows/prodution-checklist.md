# ✅ Production Checklist

## 🎯 Purpose

Provide a final validation checklist to ensure an AI skill is reliable, consistent, and ready for real-world use.

This checklist prevents deploying prompts that only work under ideal conditions.

---

## 🧠 When to Use

Use this checklist before:
- deploying a prompt to production  
- sharing a skill with users or a team  
- integrating a prompt into an application  
- finalizing a reusable AI workflow  

---

## ⚙️ Core Principle

> If it hasn’t been tested, measured, and validated — it is not production-ready.

---

## 📦 Skill Definition

- [ ] Goal is clearly defined  
- [ ] Target user is specified  
- [ ] Task is explicit and unambiguous  
- [ ] Role is clearly defined  

---

## 📏 Constraints

- [ ] All instructions are measurable and testable  
- [ ] No vague language (e.g. “good”, “clear”, “engaging”)  
- [ ] Constraints directly address known failure modes  
- [ ] No unnecessary or redundant rules  

---

## 🔒 Output Format

- [ ] Output structure is explicitly defined  
- [ ] Section order is fixed  
- [ ] Length constraints are numeric  
- [ ] No optional or undefined sections  
- [ ] Format is consistent across multiple runs  

---

## 🧪 Testing

- [ ] Tested with at least 5 inputs  
- [ ] Includes edge cases  
- [ ] Includes ambiguous inputs  
- [ ] Includes incomplete inputs  
- [ ] Includes real-world scenarios  

---

## 📉 Stress Testing

- [ ] Adversarial inputs used  
- [ ] Failures documented  
- [ ] Failure severity ranked  
- [ ] No critical failures remain  

---

## 🛠️ Debugging

- [ ] Root causes identified  
- [ ] Fixes mapped to specific failures  
- [ ] No symptom-only fixes applied  
- [ ] No regression introduced after fixes  

---

## 📊 Evaluation

- [ ] Reliability score calculated  
- [ ] All dimensions ≥ 7/10  
- [ ] Evidence provided for each score  
- [ ] No launch risks remain  

---

## ✂️ Optimization

- [ ] Weak or vague words removed  
- [ ] Redundant instructions eliminated  
- [ ] Language is precise and direct  
- [ ] Prompt is concise without losing clarity  

---

## 🔁 Feedback (Optional but Recommended)

- [ ] Feedback mechanism defined  
- [ ] Failures are logged and categorized  
- [ ] Improvements are based on real data  
- [ ] Skill works independently of feedback loop  

---

## 🔄 Consistency Check

- [ ] Same input produces same structure across runs  
- [ ] No format drift observed  
- [ ] Tone remains stable  
- [ ] Output quality does not degrade  

---

## 🚀 Final Validation

- [ ] Skill performs reliably across all tested inputs  
- [ ] Edge cases are handled gracefully  
- [ ] Outputs are reusable and predictable  
- [ ] No unexpected behavior remains  

---

## 🧠 Final Decision

### Ready for Production?

- [ ] YES → Deploy  
- [ ] NO → Return to debugging workflow  

---

## 💡 Key Insight

> Production readiness is not about “it works once”.  
> It’s about “it works consistently under pressure”.

---

## 🏁 Outcome

If all boxes are checked, you have:

- a production-ready AI skill  
- consistent and predictable outputs  
- reduced failure risk  
- a system you can trust and reuse
